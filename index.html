Problem Statement

Why Credit Card Fraud Matters

  • Fraud is rare but extremely dangerous
  • Only ~0.1% of transactions are fraudulent
  • One missed case = massive financial loss
  • Traditional methods struggle with imbalance
  • Need scalable automated systems

The Challenge

Detecting fraud in 284,807 transactions with only 492 fraud cases—an extreme class imbalance that makes automated detection critical.

Rule Justification

High Amount Rule

Fraudsters often make large purchases to maximize financial gain before the card is detected and blocked. Unusually high transaction amounts are a primary fraud indicator.

Risky Features Rule

PCA components V1-V28 capture hidden transaction patterns derived from original features. Extreme values in these components indicate anomalous behavior worth investigating.

Multi Anomaly Rule

Combining multiple weak signals creates a stronger fraud indicator than any single signal alone. When several anomalies align, fraud probability increases significantly.

Known Fraud Pattern Rule

Historical fraud data reveals recurring patterns that repeat across new fraud attempts. Matching known signatures helps catch repeat fraud strategies early.

Key Takeaway

High recall (96.14%) ensures nearly all fraud cases are caught. Low precision is acceptable due to extreme class imbalance—prioritizing fraud detection over false positives.
